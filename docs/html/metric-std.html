The metric convention
— bits canonical + per-secret readout

Every leakage measure in this study reports on one comparable scale — bits — beside a per-secret human readout (perplexity / token-F1 / recovery-rate / cosine / AUC). This page documents the convention and the talens.report layer that enforces it: a single canonical-bits extractor across all seven measure families, a legibility-preserving bits formatter, and one standardized LeakageReport row carrying both axes. It is the substrate every surface page (vec2text, bnn, resid, permutation, defenses) renders its sweep tables on.

§01

Why a shared metric

two axes, one scale

The measurement loop pairs an attack-independent probe (in bits) with a graded attack recovery on the same surface, then asks whether the two correlate across a privacy sweep. For that comparison to be legible across surfaces, every probe must speak the same canonical unit, and every recovery number must be rendered in the units a reader of that secret can interpret.

Bits canonical. MI upper bounds (CLUB), V-usable information (PVI), capacity-matched PVI, surplus description length (MDL), PID total MI, and the spectral channel-MI ceiling are all stored in bits — but they are not the same quantity. Each LeakageReport carries a bits_kind tag so a mixed table never silently conflates an MI upper bound with an SDL.

Per-secret readout. Bits alone are illegible to a reader who wants to know "how recoverable is the secret?" Beside the bits, the report renders the secret's native recovery: a token-id by top-1/top-k recovery and perplexity; text by token-F1; a permutation by recovery-rate and Kendall-τ; an embedding by cosine; membership by AUC.

The "1/100 of a bit" fix. A genuine 0.01-bit leak rounded to two decimals reads 0.00 bits — indistinguishable from no leak. The fix lives in rendering, never in the stored value: below 0.1 bit the formatter switches to millibits, so 0.01 bit reads 10.0 mbit.

Definitions & glossary

terms used on this page
TermDefinition
canonical bitsThe single comparable scalar each measure contributes to the bits axis, extracted by canonical_bits(measure, result) from the measure's output dict.
bits_kindA tag on the bits value naming which quantity it is: v_info, capacity_v_info, mi_upper_bound, sdl, pid_total_mi, channel_mi_upper_bound. Prevents conflating distinct bit-valued quantities.
per-secret readoutThe recovery axis rendered in the secret's native units. The primary field is the single graded scalar a sweep correlates bits against; the rest are human context.
millibit (mbit)1/1000 bit. The display unit below 0.1 bit so small-but-nonzero leakage stays legible.
token-F1Multiset (order-free) F1 between predicted and ground-truth token ids: harmonic mean of multiplicity-counted precision and recall. The text-recovery readout.
perplexity2^H of a per-token conditional entropy in bits — the token-id readout's "effective branching factor".
LeakageReportOne standardized row: bits (verbatim) + bits_kind + a paired Readout + sweep context (surface, layer, transform, sigma).
§03

How the layer works

measure dict → canonical bits → paired readout → LeakageReport

Each measure returns its own dict. canonical_bits pulls the headline bits scalar and tags it; the attack side computes its recovery readout separately; LeakageReport.from_measure pairs them. Because bits come from the probe dict and recovery from the attack, the layer is rendering only — it cannot manufacture a correlation between the two axes (probe ≠ attack).

Legibility contract (tested). A stored 0.01 renders 10.0 mbit, never 0.00 bits; the stored float is byte-for-byte unchanged. A declined measure (None) renders n/a, distinct from a measured zero.

§05

Conclusions

what this enables

This is infrastructure, not a leakage finding — there is no claim node here. Every downstream surface page (Tasks 2–7) renders its (bits, recovery) sweep on this layer, so the whole site reports one comparable bits axis beside a secret-appropriate recovery axis, with sub-0.1-bit leakage staying legible. Validated by 25 unit tests against the real output schema of all seven measures, with no regression across the 121-test model-free suite.
