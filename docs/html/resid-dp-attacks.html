Differential privacy
— leakage across surfaces and depth

The cross-surface view of one defense: local differential privacy, a Gaussian mechanism applied to the input embedding. The page aggregates every surface measured under DP — the residual stream observed across depth, and the pooled embedding (Bayes-NN bounds, Vec2Text). On each surface an attack-independent probe (in bits) is paired with an attack, and the budget ε is swept. The residual finding: the probes keep seeing the token while a linear ridge attack collapses and decorrelates from the probe at depth, yet a non-linear decoder re-correlates and recovers more — the gap is attack strength, not lost information.

research report · 2026-06-25
§01

Introduction

one defense, every surface it touches

Differential privacy adds calibrated noise so the released object reveals little about any one input. This page measures, surface by surface, how much of the secret survives that noise and whether an attack-independent probe predicts how much an attacker recovers. The scheme is local DP: the data owner perturbs its own input embedding before releasing anything; the noise then propagates through whatever the deployment exposes.

One scheme, observed at different depths. The only DP mechanism here is the Gaussian mechanism on the input embedding e₀ ← clip(e₀,C); e′ = e₀ + N(0,σ²I), with σ = C·z/ε. The axis is the observation depth: at L0 the released object is the noised embedding itself; at depth L>0 it is that same noise reshaped through L transformer blocks. L0 and "propagated" are not two schemes — they are one scheme seen at two depths. Central DP (a trusted aggregator adding noise to outputs) is not covered yet.

Threat model — WEIGHTS-PUB, honest-but-curious. The adversary knows the weights, the embedding table, and the published DP parameters (C, σ), and can synthesize unlimited (noised-representation, token) training pairs at any ε. Every attack compared here uses only this, so comparisons between admissible attacks are fair.

Scope. Surfaces measured under DP: the residual stream (§04) and the pooled embedding (§05). Key/value-cache and permutation-table defenses use non-DP mechanisms (KV-Cloak, GELO, permutation cover, Stained Glass) and live on their own pages. The representation-space "at-layer noise" control — Gaussian noise injected directly at the observed layer, which is not a DP deployment — has been moved to a control note (research-wiki/experiments/representation-space-noise-control.md); it is the evidence that the depth decorrelation below is caused by propagation, not by noise magnitude.

§02

Definitions & glossary

terms used on this page
TermDefinition
local DPThe Gaussian mechanism on the input embedding: clip each row to L2 ≤ C, add N(0, σ²I) with σ = C·z/ε, z = √(2 ln(1.25/δ)). Smaller ε ⇒ more noise. Sensitivity C (add/remove-to-zero adjacency).
observation depthWhich layer's residual the adversary sees. L0 = the noised embedding itself; L>0 = that noise propagated through L blocks. The secret is the token id behind the released vector.
ridge (linear attack)Closed-form linear map observation → embedding, then nearest token over the candidate pool. The information-inefficient baseline; self-generates noised training pairs.
decoder (non-linear attack)An MLP map observation → embedding (cosine loss), trained on the σ-matched noised reps (noise-aware), then nearest token. The stronger admissible attack.
CLUBVariational MI upper bound I(rep;token) in bits. Attack-independent.
V_capCapacity-matched predictive V-information in bits, read as a bounded token-identity reader that never touches the embedding table. Attack-independent.
reader perplexityThe legible companion to V_cap: 2^(H_prior − PVI) = the effective number of token candidates the reader is still choosing among (1 = fully pinned down; up to the prior's effective vocabulary = no information).
SDLSurplus description length (bits); reported auxiliary (noisier, 6–7× costlier).
recovery / selectivityrecovery = top-1 token match over the candidate pool (size 2048); selectivity = recovery − a label-shuffle floor (guards against memorization). The per-secret readout.
§03

Method

the measurement loop on the local-DP residual
FIG. 01 — local DP on the input embedding, observed at depth L · ridge / decoder vs CLUB / V_cap navy = client (forms embedding, applies DP, runs model) · plum = ε DP locus · slate = adversary (WEIGHTS-PUB) · amber = recovery

For each surface we run the measurement loop: (1) run an attack on the released representation → graded recovery; (2) run an attack-independent probe on the same representation → bits; (3) sweep the DP budget ε; (4) test whether bits and recovery correlate across the sweep. The probes never consume the attack's output, so any correlation is real, not circular. The residual sweep (§04) reads all four arms — CLUB, V_cap, ridge, decoder — from one tool, scripts/evals/dp_leakage_sweep.py, so the numbers splice into one table.

Read the plots above (the 16-row table is collapsed below). At L0 the released object is essentially the noised embedding, so the linear ridge is near-optimal and beats the non-linear decoder at every budget (0.97 vs 0.67 clean) — the non-linear attack has no edge where the map is already linear. With depth and noise the picture inverts: by L5/L12/L20 at ε ≤ 512 the decoder overtakes ridge, most sharply at L20, ε=256 (0.291 vs 0.134). Throughout, the probes barely move relative to the attack collapse: CLUB stays in the 1500–3400-bit band and V_cap's reader keeps an effective ~7–40 candidates while ridge recovery falls 3–7×. The information is preserved; the linear attack just cannot reach it at depth.

At the input layer everything tracks (ρ=1.0). At the deepest layer the linear ridge decorrelates from both probes (ρ=+0.20) while the non-linear decoder re-correlates (ρ=+1.00). This is the page's thesis in one row: the probe still sees the information at L20, the linear attack can no longer realize it across the noise sweep, and the non-linear, noise-aware decoder restores the correlation by reaching the preserved signal.

The depth-decoupling caveat (V_cap). This is the same surface where the attack-independent V_cap reader's tracking of the embedding-reconstruction attack attenuates with depth — the claim:depth-decoupling-input-dp finding (jury PARTIAL/scoped, single seed). It is consistent with the ridge column above: the linear attack and the token-identity probe move together at L0 and apart at depth. The non-linear decoder is what closes that gap.

§06

The probes

attack-independent by construction

Every probe is computed without running the attack — a probe that reported the attack's own bits would make any correlation circular. On the residual: CLUB trains a variational q(y|x) for an MI upper bound, and V_cap fits a capacity-bounded token-identity reader that never reads the embedding table; neither consumes the ridge/decoder output. On the embedding: I_G is a closed-form geometry-only ceiling and the Bhattacharyya–Fano bounds are functions of the codebook geometry and σ alone. SDL is reported auxiliary. See the probe registry for the canonical definitions.

§07

Claims

claims and their support

claim:restore-correlation: an information-efficient attack restores the probe↔recovery correlation where a weak attack decorrelates. Supported on the residual across depth: at L20 the linear ridge decorrelates from CLUB/V_cap (ρ=+0.20) while the non-linear decoder re-correlates (ρ=+1.00) and recovers more in the deep+noisy regime. Single seed, n=4 ε; directional. research-wiki/claims/restore-correlation.md

claim:bayes-gap-diagnosis: the probe↔recovery gap is an attack-efficiency gap, not poor prediction. Supported at L0 on the embedding (probe high while the exact Bayes-NN holds the ceiling and ridge collapses) and consistent on the residual (probes flat while the linear attack falls and the non-linear attack reaches more). research-wiki/claims/bayes-gap-diagnosis.md

claim:depth-decoupling-input-dp: under local DP the attack-independent V_cap reader's tracking of the embedding-reconstruction attack attenuates with depth. Scoped / jury PARTIAL, single seed; the V_cap ρ column above is the canonical example. research-wiki/claims/depth-decoupling-input-dp.md

The first two rest on the verified theorem claim:thm-t1-info-efficient (the Bayes-optimal attack weakly dominates any lossy/linear attack; recovery is monotone in MI on the Gaussian arm) — they are its empirical instantiation under DP.

§08

Conclusions

what leaks under DP, and why

What leaks. Under local DP the token information the probes measure is largely preserved (CLUB and V_cap move slowly); whether it is recovered is an attack-strength question. On the residual, the linear ridge is near-optimal at L0 but collapses and decorrelates from the probes at depth, while a non-linear, noise-aware decoder re-correlates and recovers more in the deep, noisy regime. On the embedding, a complete corrector tracks the probe without a depth gap.

The honest caveat. The non-linear decoder is not a universal winner: at L0 it underperforms the linear ridge (0.67 vs 0.97 clean) because the embedding map is already linear — its advantage is specific to depth + noise. The residual sweep is single-seed; multi-seed confidence intervals and a denser ε grid are the named firm-up.

Out of scope here. Central DP (trusted-aggregator output noise) is not yet covered; the representation-space "at-layer noise" control (not a DP deployment) is documented in the control note (research-wiki/experiments/representation-space-noise-control.md), where it establishes that the depth decorrelation is caused by propagation, not noise magnitude.
