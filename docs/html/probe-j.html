Negentropy separability
— probe J

The summed Hyvärinen negentropy of the whitened rows: how non-Gaussian, and therefore how separable by joint-diagonalization, a released key/value stack is. It is a function of whitened-row moments only, computable with the source-separation attack deleted.

probe reference · 2026-06-24
§01

Algorithm

inputs → transforms → bits

Blind source separation recovers independent sources from linear mixtures by exploiting their non-Gaussianity. J measures the resource that makes this possible, the departure of the whitened rows from Gaussian.

  1. Take the released observation stack (rows of the key/value or residual matrix).
  2. Whiten the rows: center and linearly transform so the row covariance is the identity, removing all second-order structure.
  3. For each whitened row u compute the Hyvärinen negentropy J(u) ≈ [ E G(u) − E G(ν) ]², where G is a non-quadratic contrast and ν is a standard Gaussian: the row's departure from Gaussian.
  4. Sum over rows to the separability resource J (bits). Against a matched random-demix floor it gives the genuine margin, the part above what an unstructured demix would score.

The output is in bits. Beside it the report renders the per-secret readout for the surface (source-separation recovery, or the genuine margin against the random-demix floor). The estimator is src/talens/measures/bss_separability.py; it also computes the shared spectral capacity secondary diagnostic on the same channel.

§02

Method

what it estimates · which surface · what it bounds

Kind. An independent-component separability resource: how much non-Gaussian structure the joint-diagonalization attack has to exploit. Higher J means more separable.

Surface. The key/value cache rows (key, query, value, output projections) and the residual rows, presented as an observation stack. It is defined per layer and per channel kind.

What it bounds. It tracks the blind-source-separation margin: the recovery a joint-diagonalization demix achieves above a matched random-demix floor. It ranks channel families by separability; it is not a within-channel oracle.

§03

Rationale

why attack-independent · where it tracks · where it is mismatched

Attack-independent. J is a function of the whitened-row moments only. It never runs the joint-diagonalization demix and never sees the recovered sources, so it can be computed with the separation attack deleted.

Where it tracks recovery. On the plaintext key/value accumulation baseline the genuine margin against the random-demix floor is ordered by channel kind, and J tracks it at ρ = +0.92 across the nine layer-by-kind cells. Under KV-Cloak only the channels containing the feature mix drive recovery to the floor and collapse the negentropy probe (about 1045 bits to 1.5 bits); J tracks recovery at ρ = +0.71 aggregate over 270 cells and +0.77 channel-mean, identifying the feature mix as the load-bearing channel. See the source-separation and KV-Cloak pages.

Where it is mismatched (the probe-not-matched arm). Under orthogonal GELO the loop is weak: ρ(J, genuine margin) = 0.51 at shield fraction 0 and 0.29 over all 45 cells, both below the 0.6 bar. There the leak is the published feature Gram (relative error 2.5e-16 at κ=1), a channel the whitened-row negentropy does not read. This is diagnosed as a likely probe mismatch, not a refutation; a feature-Gram-matched probe and a stronger separation suite are the queued follow-up. It is the probe-not-matched arm of the probe-failure dichotomy.
