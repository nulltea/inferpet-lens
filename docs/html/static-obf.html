Static obfuscation
AloePri, attack by attack

One defense seen through its full attack suite. AloePri covariant obfuscation (arXiv 2603.01499) is a static offline re-parameterization: secret key matrices (Algorithm 1) rewrite every weight, intra-head transforms (Algorithm 2) rotate the attention internals, a token permutation Π scrambles the wire, and embedding noise αₑ is the one additive knob, all composed so the forward output is bit-equivalent to plaintext while the server sees only obfuscated tensors. This page runs each paper attack in its own section, on Pythia-160M, under a no-key, no-paired-data threat model. The result the suite converges on: the key material is key-gated access control, not information hiding. A fixed invertible re-parameterization is absorbed exactly by an attacker who can retrain on the obfuscated basis; the only information-theoretic lever is the αₑ / ε₁ noise.

§01

Introduction

the thesis and the gate-bearing threat model

AloePri obfuscates a deployed model once, offline. A secret invertible key pair (P̂, Q̂) with P̂Q̂ = I rewrites every residual-touching weight, so the released residual is x' = x·P̂ (widened from d to d+2h) while the model output is preserved. Three further secrets compose with it: the token permutation Π, the Algorithm-2 intra-head rotations, and the embedding noise αₑ. Each section below runs one attack against this artifact and reads its token recovery.

The gate-bearing threat model. AloePri's adversary is the honest-but-curious server under Kerckhoffs's principle: it has the public plaintext model and the obfuscation algorithm, observes obfuscated tensors as a byproduct of inference, but holds no deployment key (the P̂/Q̂ seed), no token permutation τ, and no (obfuscated, plaintext) paired data from the live basis. An attack counts toward the release gate (TTRSR ≤ 0.15) only if it respects this. Two convenience baselines do not, and are reported only as ceilings: matched (a ridge inverter retrained on the deployment's own obfuscated representations with true labels, which presupposes live-basis paired data) and oracle (an inverter applied after multiplying by the true key Q̂, which presupposes the key).

Why a fixed re-parameterization is not information hiding. The key pair is a lossless change of basis: P̂Q̂ = I, so the mutual information between the obfuscated representation and the token is identical to plaintext. A linear inverter fit on the obfuscated basis recovers exactly what it would on plaintext. Protection therefore comes from withholding the basis (access control), not from destroying information. The single mechanism that reduces information is additive noise: αₑ on the embeddings and the M1 randomized-response budget ε₁ on the token wire.

§04

ISA-HiddenState, residual stream

the in-model blind attack across defence levels

The internal-state attack on the residual stream x' = x·P̂. The in-model attacker has no key and no paired data, so it uses multi-key synthesis (paper §F.1): it draws K = 64 synthetic keymats with its own seeds, fits one key-invariant ridge on the synthetic obfuscated representations, and decodes the deployment representations. The probe is the same capacity-matched V_cap (PVI) used on every surface (the function shared with the differential-privacy page), with one rule enforced: the PVI reader trains only on data the attack can also access. For no-defence the attacker reproduces the plaintext release, so the reader trains on plaintext representations; for keymat and alg1 it has no deployment-basis labels, so the reader trains on the attacker's own synthetic-key representations and is scored on the deployment representations. The probe therefore never trains on a deployment-basis labelled pair the attack could not obtain.

This corrects an earlier reading: the same probe computed in the matched regime (reader trained on deployment-basis labelled pairs) stayed flat at ~5.8 bits across all three levels and appeared to decouple from recovery. That flatness was a threat-model artifact, the probe was using data the keyless attack cannot obtain. The fix is not a different probe but a constraint on its fit pairs, the identical mechanism that keeps PVI in-model under differential privacy (there the public mechanism lets the attacker self-generate noised pairs, so the default fit is already accessible and PVI tracks recovery). AloePri's keymat is information-preserving, so a matched attacker with paired data still recovers (the information is not destroyed), but no in-model attacker, and now no in-model probe, can read it without the key. Single seed; a multi-seed replication and an explicit differential-privacy positive control are the queued firm-up.

Two-stage τ-leak cascade — does a partial harvest bootstrap the inverter?

The keyless attacker is not the only model. A frequency-analysis harvest on the token-id wire (TFMA) recovers the plain-to-obfuscated mapping τ for a subset of token TYPES; those become true labels that bootstrap a supervised inverter on any surface (the generic cascade_attack: train on the harvested types' pairs, score on the held-out types). The question is the bootstrap threshold: how many harvested types k let the attack generalize to the rest. One subplot per AloePri attack below (ridge and gradient-optimization inverters: rep → token embedding, nearest-token); within each, x = defence setting (αₑ) and the two bars are harvest size k; dotted line = release gate (0.15). Why ridge largely represents every inverter here. AloePri's defence is a linear secret basis; a k-harvest's role is to reveal that basis (k ≥ ~rank pairs least-squares-solve P̂), after which direct linear inversion is near-optimal, so a non-linear or iterative inverter adds little once a harvest is available. A GELU-MLP inverter equals ridge exactly under a harvest (table keymat 0.989/0.989), and the §F.1 transformer collapses to the ridge cell by the same argument; the paper's IMA-EmbedRow-transformer is therefore shown in its own no-harvest, τ-invariant form in §11 (not a k-harvest cell). The §D.1 gradient-optimization ISA — which optimizes a candidate input embedding by gradient descent to fit the observed residual, rather than solving the inverse in closed form — is ported here as its own subplot (ISA-HiddenState · gradient). Direct measurement refines the earlier reasoning: it does not strictly reduce to the ridge cell but tracks it to within a few points, deviating in both directions — at αₑ=0 it edges ridge (0.904 vs 0.882 at k=293), at αₑ=1 ridge edges it (0.555 vs 0.612), and at αₑ=2 they split by harvest size (gradient higher at k=293, ridge higher at k=1024). The privacy verdict is nevertheless identical to ridge's (both collapse under αₑ=2 at the realistic k=293, both still leak at k=1024), and the blind, no-harvest gradient-opt is the k=0 case.

Finding. A realistic-size clean harvest of ~293 token types (≈ the embedding effective rank, order-independent: random ≈ frequency) lets the closed-form ridge inverter bootstrap both the residual hidden state (ISA-HiddenState, keymat 0.882) and the static embedding table (IMA-EmbedRow, keymat 0.989); a non-linear GELU-MLP inverter adds nothing (these maps are affine-saturated, the table being literally (W+noise)·P̂). The §D.1 gradient-optimization inverter (ISA-HiddenState · gradient) is no different in verdict — it tracks the ridge cell to within a few points, edging it at αₑ=0 (0.904 vs 0.882, k=293) and trailing it at αₑ=1 (0.555 vs 0.612), confirming that once a harvest reveals the linear basis the inversion method is immaterial. The αₑ noise of alg1 is a sharp lever along the x-axis: at the paper αₑ=1.0 it barely helps the k=293 bootstrap (residual 0.882→0.612), and only αₑ=2.0 collapses it (0.018), so the paper default is under-parametrized for this attack. But the knee shifts with harvest size: at k=1024, even αₑ=2.0 still leaks (residual 0.532, table 0.832). Label quality is the other lever — realistic TFMA label noise (~52% clean) collapses the k=293 bootstrap (table to 0.02). The attention-value surface (kqv_out) reads ≤0.085 at every setting, including keymat where no head obfuscation is applied — but a pool-controlled check shows this is mostly a candidate-pool effect, not pure non-invertibility: the contextual attention output is still invertible to held-out types (~0.58 under a held-out-only pool, matching FIG.02's ~0.46 disjoint), but under the cascade's realistic full pool (held-out + harvested types) the map's held-out predictions resolve to already-seen tokens and collapse to ~0.08 — independent of harvest order (random 0.07 ≈ frequency 0.09) and not lifted by more harvested types. The per-token-invertible L0 residual is pool-insensitive (0.882 either way), so only this contextual surface shows the gap; Algorithm 2 is redundant for this defense either way. So AloePri's residual and table safety rests on the TFMA harvest being noisy and the clean specials harvest being low-diversity; the attention-value surface's apparent cascade-resistance is in part the wider candidate pool, not a structural wall. Oracle-τ and single-seed; the specials-only low-diversity control and a fresh kill-argument re-judge are the queued firm-ups.

§05

ISA, attention value surface (kqv_out)

the per-head value coordinates Algorithm 2 rotates

The internal-state attack reads the attention output (the per-head value-weighted sum, before the output projection) and inverts it to the token with a matched ridge, across all 12 layers. This is the surface Algorithm 2 targets with its value rotation Û_vo and inter-head permutation. Four configurations isolate the rotation from the noise: alg2@0 is the orthogonal rotation alone, alg2@1.0 adds αₑ noise.

§06

ISA, attention score surface (kq)

pre-softmax Q·Kᵀ, preserved by construction

The same internal-state attack on the pre-softmax scores Q·Kᵀ (the last 16 causal keys per head). Algorithm 2 sets M_q = M_k so the scores are preserved up to a fixed head permutation and a rotary non-commutation; Algorithm 1 leaves them unchanged. This section is the complement of §04: it shows the score surface leaks far less than the value surface, and that Algorithm 2 does not defend it against a matched attacker either.

§07

NN, cosine nearest-neighbour

training-free embedding match

The nearest-neighbour attack (paper §F.1) cosine-matches each observed residual row directly to the nearest candidate token embedding, with no training. It recovers at the input layer on plaintext (the residual is close to the embedding) but needs the observation to live in the embedding space.

§08

VMA, vocabulary matching

sorted-quantile signature on the weight rows

The vocabulary-matching attack recovers the token permutation τ by matching the sorted-quantile signature of each obfuscated embedding row to the plaintext rows. It defeats permutation-and-scaling obfuscation, where rows differ only by column permutations. AloePri's dense keymat P̂ mixes the columns, breaking the signature. The αₑ sweep (FIG.04) shows VMA flat near zero at every noise level.

§10

ArrowMatch, cosine direction

Game of Arrows, USENIX Security 2025

ArrowMatch recovers τ by cosine-matching each obfuscated weight row to its nearest plaintext direction. Its foundational observation (Obs2) is that per-vector obfuscation (a row permutation plus diagonal scaling) preserves vector direction and is therefore recoverable, while only matrix multiplication changes direction. AloePri's keymat is exactly a matrix multiplication, so it is expected immune. The sanity control is a pure token permutation (direction preserved).

§11

IMA-EmbedRow-transformer

the in-model trained inverter · measured

The paper's canonical IMA (§F.1) is a 2-layer, 8-head transformer inverter trained on the attacker's own synthetic obfuscated pairs: under Kerckhoffs the attacker picks its own pseudo-τ, Q̂, P̂ and runs the obfuscator on the public embedding table, so the inverter must learn a τ-invariant inverse without any deployment leak. We port that architecture (`talens.attacks.dp_inversion.IMAInverter`) and train it on K = 4 synthetic own-key obfuscations of the table, then decode the deployment table's held-out rows. A matched plain control (train and test on the same deployment key, row-split) verifies the inverter actually inverts.

Finding. The in-model τ-invariant transformer recovers ≈ 0 at every defence level, including keymat with no noise — so AloePri defends against the canonical IMA, and the defense is the keymat (basis hiding), not the αₑ noise: no single inverter inverts the random per-key change of basis it never saw. The plain control confirms this is a real defense, not a broken attack: the same inverter recovers 0.998 when trained and tested on one key (and αₑ noise degrades even that matched case, 0.998 → 0.915 → 0.121, the same αₑ lever seen elsewhere). This reproduces the paper's reported IMA ≈ 0 % on a faithful port that passes the identity-key control (the companion deployment study's driver did not). The inverter is data-hungry at d = 768: the control needs ample training rows (≈12k tokens, 40 epochs); small sets underfit to ≈ 0 and are not interpretable.

TFMA, frequency matching

the token-id wire, ε₁ sweep

On the wire surface the server sees the Π-permuted, M1-perturbed token stream. TFMA matches the obfuscated-stream unigram frequencies to a reference to undo Π. Π alone is a substitution cipher (recoverable); the M1 randomized-response budget ε₁ is the lever that suppresses recovery. The analytic channel mutual information tracks TFMA recovery across the sweep (Spearman 1.0).

§15

Conclusions

what AloePri protects, and how

Key-gated access control, not information hiding. AloePri's key material is a lossless re-parameterization, so it preserves mutual information and is absorbed by any attacker who can retrain on the obfuscated basis. The residual and value surfaces show this directly: the matched inverter recovers the token nearly fully while the in-model multi-key blind stays under the gate, and Algorithm 2's value rotation gives identical recovery to plaintext under a matched attacker (plaintext, keymat, and Algorithm 2 without noise coincide at every depth on both kq and kqv_out).

The single information-theoretic lever is noise. The only mechanism that reduces recovery is additive noise: αₑ on the residual and value surfaces (value recovery at L0 falls from 0.685 to 0.564 row-split, 0.462 to 0.179 disjoint) and the M1 ε₁ budget on the wire (TFMA 0.52 to 0.03, SDA 0.75 to 0.00). Everything else is access control: defeated by the keymat structure (VMA, Attn-IA, ArrowMatch, NN), defeated only by the noise (Gate-IA), or not defended against a key or paired-data attacker (matched ridge, oracle).

Scope and caveats. Single seed, Pythia-160M, 160 prompts. The matched score and value readings are linear-probe leakage measurements, not deployable attacks (they presuppose paired data). The in-model IMA-EmbedRow-transformer is now measured (τ-invariant recovery ≈ 0 at every αₑ; plain control 0.998), reproducing paper IMA ≈ 0% on a port that passes the identity-key control. The weight and wire attacks reproduce the companion Qwen3-4B deployment study at the analogous configuration.
