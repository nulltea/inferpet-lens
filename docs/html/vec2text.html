Vec2Text
— embedding inversion

Faithful pretrained Vec2Text (Morris et al., EMNLP 2023) as a confidential-inference attack: recover source text from a released sentence embedding. Studied on two surfaces — it fails on the per-position residual stream (no bottleneck) and succeeds on a pooled GTR retrieval embedding. The matched, attack-independent leakage probe is the spectral channel-MI I_G(σ); the privacy–utility tradeoff is charted against retrieval fidelity.

§01

What the attack is

threat model & the two surfaces

Vec2Text inverts a dense text embedding back to its near-exact source text. Given a released embedding e and query access to the encoder φ, it iteratively generates a hypothesis text, re-embeds it, and corrects toward e — recovering ~92% of 32-token inputs exactly in the original paper. It is the inversion attack that motivates protecting released embeddings (RAG vector stores, split-inference activations).

Threat model — WEIGHTS-PUB-analog for embeddings. The adversary holds the public encoder φ (here GTR-base), the published DP parameters (clip C, noise σ), and the matched pretrained corrector; it observes the DP-protected released embedding e′ and must recover the text. The DP defense is an external Transform on the released vector — the library asserts nothing about it (scheme-agnostic).

Two surfaces, opposite verdicts. Vec2Text exists to invert a single pooled bottleneck vector; whether it applies depends entirely on the released object.

SurfaceReleased objectVec2Text verdict
Per-position residual streamresid_post[L] — one d-dim vector per tokenWRONG TOOL. Each token has its own observation (logit-lens-readable; LMs are per-position invertible, arXiv 2510.15511). No bottleneck ⇒ the iterative feedback is moot — empirically disproved (§07, B7).
Pooled sentence embeddingGTR mean-pooled — one 768-d vector for the whole textCORRECT TOOL. The whole sequence is compressed into one bottleneck ⇒ the under-determination Vec2Text resolves is real. Recovers text; the surface this report studies.

§03

How the attack works

pipeline · DP defense · iterative corrector loop
FIG. 01 — pipeline · text → pooled embedding → DP release → Vec2Text inversion navy = data owner (trusted) · plum = DP defense · slate = adversary · amber = leak

The iterative corrector (steps ① ②) is the mechanism. A base seq2seq model guesses an initial hypothesis h₀ from e′. Each round re-embeds the current hypothesis ê⁽ᵗ⁾=φ(h_t) and the corrector — conditioned on the target e′, the hypothesis embedding ê⁽ᵗ⁾, the residual e′−ê⁽ᵗ⁾, and the hypothesis tokens — emits a new sequence h_{t+1}, accepted only if its re-embedding is closer to e′. The residual e′−ê⁽ᵗ⁾ is the fresh information injected each round; it is what makes the loop converge (Morris Fig. 1: ê⁽⁰⁾→ê⁽¹⁾→ê⁽²⁾→ê⁽³⁾ spiralling onto e).

Why the bottleneck matters. The loop exists to resolve the under-determination of compressing a whole sequence into one vector. On a pooled embedding that compression is real, so iteration pays off. On the per-position residual stream there is one observation per token (no compression), the feedback is moot, and a one-shot per-position decoder already suffices — which is exactly what the per-position experiment found (§07, B7).

Probes — measuring leakage without running the attack

capPVI · CLUB · spectral channel-MI (matched)

A probe estimates how much the released embedding leaks about the text, from data geometry + channel parameters alone — never running an inverter (the "not a disguised attack" invariant). Two incumbents are inadequate; the matched one is I_G.

inadequate

capPVIcluster V-information

Flat / orthogonal. Predicts a κ-means cluster label of e₀ from e′. Saturates at ≤log₂κ ≈ 5.3 bits and stays flat while σ ≪ centroid gaps — a coarse, noise-robust quantity on a scale orthogonal to fine token recovery.

Targets the right MI but non-localizing. By the sufficiency identity (T1) I(e′;e₀)=I(X;Y) — the true leakage. But CLUB is a variational upper bound with estimator variance and a single scalar: monotone in σ, yet no per-mode structure.

Proofs — the spectral channel-MI probe

claim:spectral-channel-mi-embedding-inversion · Codex xhigh · 3 rounds · PASS

Setting: secret text X (finite alphabet), deterministic clipped embedding e₀=clip(φ(X),C) (discrete), release Y=e₀+N(0,σ²I), Σ=Cov(e₀) eigenvalues λᵢ. The continuous-channel analog of the discrete Bhattacharyya–Fano bound (text has Vⁿ messages — no enumerable codebook).

ThmStatementRole
T1I(X;Y) = I(e₀;Y) — deterministic sufficiency identity.measuring MI on embeddings is text leakage (and is CLUB's target).
T2I(X;Y) ≤ min{H(e₀), I_G(σ)}, strict unless e₀ Gaussian (max-entropy). Per-mode tᵢ=½log₂(1+λᵢ/σ²), d_eff=#{λᵢ≥σ²}.closed-form bit ceiling + localization (water-filling).
T3Fano: uniform-prior P(x̂=X) ≤ (min{H(e₀),I_G}+1)/H(X) for every attack; rate–distortion floor on the per-token error.attack-independent recovery ceiling (converse), monotone in σ.
T4For top-k principal projector P_k (arbitrary e₀): I(X;Y) − I(X;P_kY) ≤ ½Σ_{i>k} log₂(1+λᵢ/σ²).leakage is confined to the top eigendirections up to the spectral tail — "where".
ContrastcapPVI ≤ min{I(X;Y), log₂κ} (saturates); CLUB is a loose, non-localizing variational bound on the same target.proves why the incumbents fail.

Honest scope. I_G is a converse ceiling, not exact leakage nor achieved recovery — the empirical probe↔recovery correlation (§07) is validation, not implied. The low-noise cap is the discrete H(e₀); token-F1 (vs positional error) and T4 achievability are flagged as empirical hooks. Imported textbook steps: Gaussian max-entropy, Fano, the Shannon rate–distortion lower bound. Full proof: refine-logs/dp-stronger-attacks/vec2text-pooled/PROOF_PACKAGE.md.

Results

gemma-2-2b (per-position) · GTR-base (pooled) · AMD ROCm iGPU

B7 — per-position residual stream · the negative

The faithful iterative corrector (re-embed + residual feedback + acceptance gate) was implemented on the per-position resid surface. Feedback did not help (feedback ≤ no-feedback at every ε) — the bottleneck the loop needs does not exist there. Confirmed against the reference (vec2text/vec2text = pooled embeddings only) and Rep2Text (single last-token rep). Verdict: wrong surface → pivot to pooled.

§08

Conclusions

what is established

1 · Surface decides the attack. Vec2Text is a bottleneck-inversion attack — correct on a pooled sentence embedding, wrong on the per-position residual stream (where logit-lens/probing is the matched tool). The feedback loop is only useful where a whole sequence is compressed into one vector.

2 · The pooled embedding leaks, DP defends it cheaply. Faithful Vec2Text recovers real text from clean GTR embeddings; the Gaussian mechanism on the released vector defeats exact reconstruction at a small budget, and — crucially — retrieval utility survives that budget (favorable privacy–utility curve).

3 · The matched probe is geometry, not an attack. The spectral channel-MI I_G(σ) predicts SOTA recovery as well as CLUB at ~28× lower cost, is a verified converse ceiling on any attack, and uniquely localizes leakage to the top embedding eigendirections — replacing the flat capPVI and the loose, non-localizing CLUB.

Open. Localization (T4) achievability via eigen-ablation needs Σ estimated at n≫d; real BEIR/MTEB qrels would upgrade the utility ground truth from the clean-ranking proxy; the per-token-F1 vs positional-error gap stays an empirical hook.
