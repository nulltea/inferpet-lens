Surplus description length
— probe SDL

The prequential online code length of the representation-to-label map, minus the all-data floor: the extra bits paid to learn the map online. A complexity-aware leakage signal that is auxiliary to the cheaper predictive readers.

probe reference · 2026-06-24
§01

Algorithm

inputs → transforms → bits

Minimum-description-length codes a label sequence by transmitting it online: each label is encoded with the model fit on the labels seen so far. A more decodable representation pays fewer bits.

  1. Order the examples and present the representation-to-label pairs as a stream.
  2. For each prefix, fit the reader on the seen pairs and predict the next label; accumulate its negative log-probability. The total is the prequential (online) code length.
  3. Subtract the all-data floor (the code length of the same labels under the model fit on the full data): the difference is the surplus description length SDL, the extra bits the online learner pays over the in-hindsight optimum (Whitney SDL).
  4. Report SDL in bits, with the shuffle-control floor subtracted to give selectivity.

The output is in bits. The estimator is src/talens/measures/mdl.py. It inherits the same reader family as the capacity-matched predictive reader, and a single online pass costs roughly 6 to 7 times a single bound evaluation, so it is reported as an auxiliary, complexity-aware signal rather than a primary probe.

§02

Method

what it estimates · which surface · what it bounds

Kind. A complexity-aware code length (minimum description length / surplus description length). It charges for the effort of learning the representation-to-label map online, not just the final fit, so a representation that is easy to read incrementally scores low.

Surface. The residual stream with a token-identity label, the same surface as the capacity-matched predictive reader. It is layer-defined.

What it bounds. The surplus bits index how decodable the token is, accounting for sample complexity. It is a leakage signal rather than a converse bound; it is reported beside the predictive reader as a complexity-aware cross-check.

§03

Rationale

why attack-independent · where it tracks · its cost

Attack-independent. The prequential code length is computed from the reader's own predictions on held-out prefixes; it never runs the inversion attack and never reads the embedding table. Its shuffle-control floor isolates the genuine signal from the label prior.

Where it tracks recovery. Under at-layer Gaussian noise at layer 12, the surplus selectivity falls from 13898 bits at zero noise to 166 bits at the strongest level and tracks the ridge recovery at Spearman ρ = +0.80. It is one of the predictive-family probes that re-correlate with a stronger attack in the at-layer regime.

Its cost and standing. A single online pass costs roughly 6 to 7 times a single bound evaluation, and the probe inherits the same reader-family sensitivity as the capacity-matched predictive reader. Because the project minimizes probe cost, surplus description length is kept as an auxiliary complexity-aware signal rather than a primary probe.
